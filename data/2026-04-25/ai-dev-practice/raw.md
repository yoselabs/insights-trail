**Fetched content (Anthropic context engineering for agents):**
URL: https://www.anthropic.com/engineering/effective-context-engineering-for-ai-agents
- Context engineering = optimizing token allocation in limited context windows
- "Find the smallest set of high-signal tokens that maximize the likelihood of your desired outcome"
- System prompt Goldilocks zone: specific enough to guide, flexible enough to heuristics
- Organize with XML tags or Markdown headers
- Keep toolsets lean to avoid ambiguous decision points
- Few-shot examples: curate diverse canonical examples
- Compaction: summarize conversation history, reinitiate with compressed context
- Structured Note-Taking: agents maintain persistent memory files outside context window
- Sub-Agent Architectures: specialized agents return condensed summaries (1,000–2,000 tokens) to coordinator
- "Just in time" runtime context retrieval: agents dynamically load data via tools rather than pre-processing
- Context rot and mode collapse: two fundamental challenges in 2026

**Key extracted facts:**
- RAG evolving into "Context Engine" — from specific pattern to broader architecture
- Context engineering: intentionally designing every slot in the LLM context window
- RAG is one retrieval primitive within a larger context engineering system
- "Retrieval-first, long-context containment" synergy drives context engineering emergence
- Context rot: performance degrades as context windows grow with poorly curated information
- Mode collapse: output diversity reduced through alignment training
- Similarity-based retrieval risky for safety procedures, regulatory policies, escalation protocols
