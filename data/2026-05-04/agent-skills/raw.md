### Query 8: Anthropic Engineering Blog — Agent Skills Architecture

URL: https://www.anthropic.com/engineering/equipping-agents-for-the-real-world-with-agent-skills

**Three-Level Progressive Disclosure:**
1. Metadata level: Name and description loaded at startup into system prompt
2. Core content: Full SKILL.md loaded when Claude determines relevance
3. Referenced files: Additional markdown and code files accessed only as needed

**Skill Structure:**
- A skill is a directory containing a SKILL.md file
- SKILL.md begins with YAML frontmatter containing `name` and `description`
- Additional files (like `reference.md`, `forms.md`) can be bundled and referenced

**Technical Implementation:**
- Claude invokes bash tools to read skill files, selectively load supplementary materials, execute included scripts
- Code can serve as both executable tools and documentation
- Security: install only from trusted sources; audit bundled code and external network connections

### Query 9: MCP vs CLI Skills Evaluation — Arize AI

URL: https://arize.com/blog/mcp-vs-cli-skills-for-agents-what-our-eval-found-and-which-you-should-use/

**Performance Metrics:**
- Correctness (all roughly equal): LobeHub (0.826), Vault (0.833), MCP (0.834)
- Cost & Latency (Tier 4 - Complex Tasks): MCP cost 6x more and took 5x longer than skills-based
- MCP: avg 12 tool calls/task; Skills: avg 5 calls/task
- Example: One task cost MCP $2 (71 calls, 8 min) vs Vault $0.19 (7 calls, <1 min)
- Tool fidelity: Skills 99%+ adherence; MCP only 0.33 on complex tasks (resorted to bash)

**Critical Discovery:**
- Bare Claude with no tooling nearly matched skill performance (0.845 correctness) for widely-known tools
- For proprietary/uncommon tools, skills dramatically outperform bare Claude

**Conclusion:**
- Use CLI tools for local work with pre-configured authentication
- Use MCP for remote, proprietary, consumer-facing applications requiring OAuth
