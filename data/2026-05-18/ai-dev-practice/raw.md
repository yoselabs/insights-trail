**Key data extracted:**
- 8 LLM failure modes: prompt fragility, retrieval degradation, hallucination, latency, agent safety, guardrails, observability gaps, cost governance
- Multi-agent error propagation: one wrong intermediate result cascades confidently through the chain
- Context window mismanagement → silent failures: model loses instructions without warning
- Factual confabulation: RAG fails at retrieval and at generation (model overrides retrieved context)
- Instruction drift: factually coherent but structurally non-compliant outputs break downstream parsing
- "Two identical requests can produce different answers, different tool usage, different reasoning paths"
- "Availability without quality is not reliability for AI"
- Most reliability failures are operational gaps, not model gaps
- Four reliability dimensions: availability, quality, detectability, recoverability
- Critical: daily evals (not weekly), drift detection, automated rollback, structured incident response

- Diagnostic: low Context Precision → fix retrieval; low Faithfulness → fix prompt or add guardrails
